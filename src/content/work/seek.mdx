    <h2>The Problem</h2>
    <p>Seek AI 1.0 asked business users to write SQL to get an answer out of their own data — a tool built for engineers, sold to executives. Steep learning curve, no way to verify AI-generated answers, and nothing built for teams.</p>
    <p>Seek needed to become a natural-language product fast enough to pitch investors and land partners like Snowflake. Four months, start to ship.</p>
  </div>
</section>

    <h2>Insights</h2>
    <p>Every interview surfaced the same workaround: ask ChatGPT to write the SQL, paste it into Seek, hope it runs. Users had already solved the interface problem — with a different product.</p>
    <p>That reframed everything: Seek needed to be conversational, not transactional. A person you ask, not a tool you operate.</p>
